# Criterios de Evaluacion - Ejercicio 4: PokeDex Avanzado

## Puntuacion Total: 100 puntos

---

### Tarea 1: Buscador avanzado (20 puntos)

| Criterio | Puntos |
|----------|--------|
| Busqueda por nombre funciona correctamente | 5 |
| Busqueda por ID funciona correctamente | 3 |
| Se muestran resultados con sprite, nombre y tipos | 5 |
| Manejo de errores (Pokemon no encontrado, entrada invalida) | 4 |
| Interfaz limpia y facil de usar | 3 |

### Tarea 2: Filtros (20 puntos)

| Criterio | Puntos |
|----------|--------|
| Filtro por tipo funciona correctamente | 5 |
| Filtro por generacion funciona correctamente | 5 |
| Filtro por rango de estadisticas funciona | 5 |
| Se pueden combinar multiples filtros | 3 |
| Los resultados se actualizan correctamente al cambiar filtros | 2 |

### Tarea 3: Vista de detalle (20 puntos)

| Criterio | Puntos |
|----------|--------|
| Muestra sprite grande (artwork oficial) | 2 |
| Grafico de estadisticas (barras o radar) | 5 |
| Tabla de debilidades y resistencias de tipo | 5 |
| Cadena evolutiva con sprites | 5 |
| Descripcion del Pokemon (flavor text) | 3 |

### Tarea 4: Comparador de Pokemon (20 puntos)

| Criterio | Puntos |
|----------|--------|
| Se pueden seleccionar 2-3 Pokemon para comparar | 4 |
| Grafico superpuesto de estadisticas | 6 |
| Tabla comparativa (tipos, habilidades, peso, altura) | 5 |
| Indicacion de ventaja de tipo entre los Pokemon | 5 |

### Tarea 5: Constructor de equipo (20 puntos)

| Criterio | Puntos |
|----------|--------|
| Seleccion de hasta 6 Pokemon para el equipo | 4 |
| Analisis de cobertura de tipos | 5 |
| Debilidades y resistencias combinadas del equipo | 5 |
| Estadisticas promedio del equipo | 3 |
| Sugerencias para mejorar el equipo | 3 |

---

## Criterios Generales (aplicados a todas las tareas)

Estos criterios pueden sumar o restar puntos sobre el total:

| Criterio | Efecto |
|----------|--------|
| Codigo bien organizado y legible | +5 puntos |
| Uso correcto del cache (no peticiones innecesarias) | +5 puntos |
| Comentarios claros en espanol | +3 puntos |
| Manejo robusto de errores en toda la aplicacion | +5 puntos |
| Interfaz visualmente atractiva y funcional | +3 puntos |
| Codigo duplicado o desorganizado | -5 puntos |
| La app se cuelga o muestra errores no manejados | -5 puntos |
| No usa los modelos proporcionados (trabaja con dicts crudos) | -5 puntos |

## Niveles de Logro

| Puntuacion | Nivel | Descripcion |
|------------|-------|-------------|
| 90-121 | Excelente | Dominio completo del consumo de APIs y construccion de interfaces |
| 70-89 | Bueno | Buen manejo de APIs con areas de mejora en la interfaz |
| 50-69 | Suficiente | Funcionalidad basica lograda, falta pulir detalles |
| 30-49 | En desarrollo | Necesita reforzar conceptos de APIs y/o Streamlit |
| 0-29 | Inicial | Requiere apoyo adicional con los fundamentos |
