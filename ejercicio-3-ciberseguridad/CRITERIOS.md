# Criterios de Evaluacion - Laboratorio de Ciberseguridad

## Puntuacion Total: 100 puntos

---

## 1. Identificacion de vulnerabilidades (25 puntos)

| Criterio | Puntos |
|----------|--------|
| Identificar correctamente cada vulnerabilidad presente | 5 pts c/u |
| Clasificar correctamente la severidad (Alta/Media/Baja) | 2 pts c/u |
| Relacionar con la categoria OWASP Top 10 correspondiente | 1 pt c/u |

**Nota:** Se espera que el estudiante encuentre al menos 4 vulnerabilidades distintas.

---

## 2. Scripts de deteccion (20 puntos)

| Criterio | Puntos |
|----------|--------|
| Cada script se ejecuta correctamente sin errores | 4 pts |
| Cada script detecta la vulnerabilidad de forma automatica | 4 pts |
| Cada script muestra evidencia clara del hallazgo | 4 pts |
| El codigo esta limpio y bien comentado | 4 pts |
| Los scripts usan buenas practicas (manejo de errores, etc.) | 4 pts |

---

## 3. Scripts de explotacion (20 puntos)

| Criterio | Puntos |
|----------|--------|
| Cada script explota la vulnerabilidad exitosamente | 5 pts |
| Se demuestra el impacto real de la vulnerabilidad | 5 pts |
| El script incluye explicacion paso a paso en comentarios | 5 pts |
| La explotacion es controlada (no destructiva) | 5 pts |

---

## 4. Aplicacion parcheada (20 puntos)

| Criterio | Puntos |
|----------|--------|
| Cada vulnerabilidad esta correctamente corregida | 5 pts |
| La aplicacion sigue funcionando correctamente despues de los parches | 5 pts |
| Se usan buenas practicas de seguridad en las correcciones | 5 pts |
| Cada correccion incluye comentario explicando el cambio | 5 pts |

---

## 5. Reporte de seguridad (15 puntos)

| Criterio | Puntos |
|----------|--------|
| Formato profesional y uso de la plantilla proporcionada | 3 pts |
| Descripcion clara de cada vulnerabilidad | 3 pts |
| Pasos de reproduccion detallados y replicables | 3 pts |
| Evidencia incluida (capturas, respuestas del servidor, etc.) | 3 pts |
| Recomendaciones de remediacion claras y correctas | 3 pts |

---

## Bonificaciones (+10 puntos posibles)

| Criterio | Puntos |
|----------|--------|
| Encontrar vulnerabilidades adicionales no obvias | +3 pts |
| Scripts particularmente elegantes o creativos | +2 pts |
| Recomendaciones de seguridad adicionales (headers, HTTPS, etc.) | +3 pts |
| Implementar tests automatizados para verificar los parches | +2 pts |

---

## Penalizaciones

| Criterio | Penalizacion |
|----------|-------------|
| Scripts que no se ejecutan | -5 pts c/u |
| Parches que rompen funcionalidad | -5 pts c/u |
| Reporte incompleto o sin evidencia | -5 pts |
| No seguir la estructura de entrega solicitada | -3 pts |
